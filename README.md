# Personal Site

A small static website for public writing.

The important parts:

- `content/inbox/` is where raw thoughts go. Messy is fine.
- `content/posts/` is the publishable layer. Codex turns inbox material into these files.
- `public/` is generated by `npm run build`. Do not edit it directly.

## Dumping thoughts

Create or append to a dated inbox file:

```text
content/inbox/2026-06-16.md
```

Use whatever shape is fastest: bullets, paragraphs, fragments, pasted notes, links, or rough titles. If a file contains multiple ideas, separate them with:

```markdown
---
```

## Publishing workflow

When you want something on the site, say something like:

```text
Take the latest website inbox and publish the strongest piece.
```

or:

```text
Turn the section about <topic> in the website inbox into a post.
```

Codex should then:

1. Read the relevant file in `content/inbox/`.
2. Draft or clean the piece into `content/posts/<slug>.md`.
3. Keep the original inbox text intact.
4. Run `npm run build`.
5. Tell you what changed.

## Commands

```bash
npm run build
npm run dev
npm run check
```

`npm run dev` builds the site and serves `public/` locally.

## Public deployment

Use GitHub Pages for a public non-Vercel deployment. Publish this folder as
its own GitHub repository; do not publish the whole Atelier workspace.

Important:

- A clean URL like `yourname.com` requires buying or already owning that domain.
- Without a custom domain, GitHub Pages uses a `github.io` URL.
- This user site should use a repo named `chaosconstruct.github.io`.
- A project site can use any repo name, but the default URL becomes
  `https://chaosconstruct.github.io/<repo-name>/`.

First deployment from this folder:

```bash
git init
git add .
git commit -m "Initial personal site"
git branch -M main
git remote add origin git@github.com:chaosconstruct/chaosconstruct.github.io.git
git push -u origin main
```

On GitHub, open the repository settings, go to `Pages`, and set the source to
`GitHub Actions`. The included workflow builds the site and deploys `public/`.

For a custom domain, add the domain in `Settings > Pages > Custom domain`, then
configure DNS with your domain registrar. For an apex domain like `example.com`,
use GitHub Pages `A` records. For a subdomain like `www.example.com`, use a
`CNAME` pointing at `chaosconstruct.github.io`.
